# Family Assets 專案交接與決策紀錄

最後更新：2026-06-24

## 專案定位

家庭資產管理 App，現為 Vercel 網頁草稿，未來預計以 PWA／Capacitor 方式測試並上架行動平台。特色方向為「島嶼＋怪獸居民培養」，讓資產紀錄與健康財務行為具有養成感。

## 已確認的產品決策

- 島嶼代表目前家庭財務狀態；怪獸代表已完成的財務歷程與永久里程碑。
- 怪獸達成後不可因市場下跌而退化或消失。
- 個人帳戶可自由命名，名稱只作顯示，不可作資料識別碼。
- 免費規劃：一個個人帳戶與一個「總資產」聚合視圖；其他帳戶未來作為付費解鎖。
- 「總資產」是聚合視圖，不是實際持有資產的帳戶。
- 主要養成來源：個人／總資產、淨資產相較起點的成長、資產種類、配息／利息、持續使用習慣與財務目標。
- 負債只保留簡化的紀念里程碑，例如償還一半、單筆清償與總負債歸零，不作主要進化線。
- 遊戲起始基準從正式上架後、使用者首次啟用時開始建立，不追溯草稿期測試資料。
- 目前歷史圖是依現有持倉與市場歷史報價重建，並非每日帳戶快照；無法完整還原已賣出持倉、歷史現金、歷史負債及不動產估值。

## 下一階段建議順序

1. 通用帳戶 ID 與聚合視圖資料模型。
2. 為資產、負債、交易、配息等資料補穩定 ID 與 schemaVersion。
3. 建立遊戲首次啟用基準、怪獸選擇及永久成就紀錄。
4. 建立第一版里程碑判斷引擎。
5. 再製作島嶼、怪獸孵化／進化及里程碑圖鑑 UI。

## 安全移轉檢查清單

每次搬移專案、換電腦、公開儲存庫、交接或正式上架前，必須逐項確認：

- 撤銷所有曾在聊天、截圖、命令列輸出或明文檔案中出現過的 Token／API Key。
- 重新建立最小權限、限定儲存庫且有合理期限的憑證。
- 使用 Git Credential Manager 或平台安全秘密管理，不使用明文 `.git-credentials`。
- 搜尋 Git 工作目錄與歷史，確認沒有秘密、個人財務資料、真實測試資料或憑證。
- 將伺服器秘密放入部署平台的環境變數，不放進前端 JavaScript。
- 檢查 Vercel、GitHub、Apple、Google、行情供應商等帳號的權限與離職／交接存取。
- 正式上架前建立隱私政策、資料刪除流程、備份／還原機制及行情來源授權檢查。
