## 使用者已確認的產品方向

### 遊戲化核心概念

使用者決定用「島嶼 + 孵化生命 / 守護獸 / 怪獸」作為資產紀錄的遊戲化主題。

目前共識：

- 島嶼代表目前財務狀態。
- 孵化生命、守護獸或怪獸代表里程碑、習慣、重要事件。
- 不要只用資產規模決定等級，避免一開始輸入大量歷史資產就直接跳到最高等。
- 里程碑可以有：個人、家庭總資產、淨資產、資產多元化、配息 / 利息、持續紀錄、財務目標等。
- 負債里程碑簡化，只做紀念型或鼓勵型，例如還款過半、清償，不作為主要養成壓力來源。

### 初始建檔 / 島嶼前傳

使用者已同意以下規則：

1. 初次使用時可以輸入過去資產、負債、交易、配息等歷史資料。
2. 初始建檔期間不重複播放解鎖動畫，也不把舊資料當成每日連續紀錄。
3. 使用者手動按「完成初始建檔」後，系統才建立 baseline。
4. 完成初始建檔後，如果補輸入更早的歷史資料，不自動重播全部動畫，只提示「島史新增片段」，可讓使用者手動觀看。
5. 更正舊資料不應刪除已獲得成就，除非未來特別設計重算機制。

### 帳戶與名稱注意事項

目前 app 內部仍有 `me`、`wife`、`family` 等概念，但未來使用者名稱可以自訂，例如可以叫「多拉A夢」。

未來規劃：

- 免費版：一個個人帳戶，可自由命名 + 一個總資產視圖。
- 其他個人帳戶可能作為付費解鎖。
- 程式設計上不要把顯示名稱當成固定 ID。
- 遊戲資料應盡量使用穩定 ID / scope，不要硬綁「我 / 太太」文字。

### 遊戲化底層規格

核心方向定版為「資產生命誌 / Asset Chronicle」。這不是單純的成就徽章系統，而是「自然生態 + 航誌 + 孵化」的財務生命紀錄。第一版遊戲化入口定為「島嶼蛋」：完成初始建檔後取得一顆蛋，後續財務行為產生不同養分，養分組合決定孵化方向。孵化生命、守護獸或怪獸不是一開始固定，而是由養分與島嶼區域自然長出來。

命名與世界觀原則：

- 對外與核心概念避免使用「財富島、共富島、資產島、理財島、金錢島」這類直白名稱。
- 目前定版主軸使用「資產生命誌 / Asset Chronicle」。
- 內部仍可保留島嶼、棲地、島史、航誌、年輪、蛋、養分、守護獸、怪獸、未知生命等元素。
- 歷史人文可以作為文案風格、主題包或華語版彩蛋，但不綁死底層規則。
- 多語言策略後續再細化；目前先避免使用過度依賴單一文化典故的底層名稱。

重要平衡原則：

- 不獎勵高頻操作本身，避免讓每天記帳或頻繁交易的人天然比較強。
- 不懲罰低頻使用者，月度整理、季度回顧、年度盤點也應能推進。
- 不鼓勵為了解鎖而交易；買賣可以進島史，但交易次數不是主要成長來源。
- 長期持有、低交易頻率、穩定盤點，也要有完整成長路線。
- 初始建檔只是島嶼前傳，不把大量舊資料一次轉成大量解鎖。
- 負債只做正向紀念與管理鼓勵，不做扣分、羞辱或壓力型怪物。
- 無配息、無負債、資產種類較少的使用者，也不能被卡住主線。

島嶼成長採多軸線，而不是單一路線：

| 軸線 | 島嶼區域 | 代表意義 | 適合的使用者行為 |
|---|---|---|---|
| 盤點軸 | 起始海灘 | 看清楚財務起點與島史 | 初始建檔、補登、整理舊事件 |
| 紀錄軸 | 記帳森林 | 穩定回來整理，不要求每天 | 月度整理、季度回顧、重要事件記錄 |
| 成長軸 | 成長山丘 | 相對 baseline 的淨值改善 | 長期持有、儲蓄、資產成長 |
| 配置軸 | 配置港口 | 資產結構逐漸清楚 | 分類、配置、再平衡，不強迫多買 |
| 現金流軸 | 收成果園 | 配息、利息、租金等收入流 | 記錄現金流；非必修主線 |
| 減壓軸 | 減壓溫泉 | 負債被看見、管理與下降 | 還款進度、單筆清償、總負債歸零 |

島嶼蛋與養分系統：

- 完成初始建檔後，使用者取得第一顆 `first_island_egg`。
- 初始建檔資料只形成島嶼前傳，不直接把蛋孵出來。
- 完成建檔後的新紀錄、整理、成長與回顧會給蛋不同養分。
- 蛋的狀態建議分為：`sleeping`、`glowing`、`cracking`、`hatched`。
- 不要求使用者取得所有養分；累積多種養分即可推進，避免只偏向某一類財務行為。
- 第一版可先用「累積任意 3 種養分讓蛋裂開、累積一定養分點數後孵化」作為方向，實際門檻後續再微調。

養分類型：

| 養分 | 來源 | 平衡方式 | 可能孵化傾向 |
|---|---|---|---|
| 記錄養分 | 完成建檔後的新月份整理、重要事件紀錄 | 一個月份最多推進一次，不做每日打卡壓力 | 森林系生命、記錄型特徵 |
| 回顧養分 | 查看並收進島史片段、整理補登事件 | 多筆補登可進島史，但不靠大量補登刷進度 | 島史系生命、回憶型特徵 |
| 成長養分 | 淨值相對 baseline 改善 | 看階段與穩定度，不看短期行情跳動 | 山丘系守護獸幼體 |
| 配置養分 | 資產分類清楚、配置結構建立、使用再平衡 | 不鼓勵為了解鎖亂買，只看結構是否更清楚 | 港口系領航員、航海型角色 |
| 收成養分 | 配息、利息、租金等現金流紀錄 | 非必修主線，沒有現金流策略也不會卡住 | 果園系生命、收成型特徵 |
| 減壓養分 | 負債被納入管理、下降、清償 | 只做正向鼓勵，不做壓力或扣分 | 溫泉系生命、安定型守護獸 |

孵化方向：

- 主要養分不直接等於固定角色，而是決定孵化傾向。
- 記錄養分較高：偏森林系，可能是書記、記錄員、觀察者。
- 回顧養分較高：偏島史系，可能是史官、收藏者、回憶守護者。
- 成長養分較高：偏山丘系，可能是守護獸幼體或拓地型角色。
- 配置養分較高：偏港口系，可能是領航員、航線管理者。
- 收成養分較高：偏果園系，可能是照料員、採收者。
- 減壓養分較高：偏溫泉系，可能是管家、安定型守護獸。
- 若養分均衡，角色可走「混合型島嶼生命」或「混合型守護獸幼體」方向，避免系統硬把使用者貼單一標籤。

升級、轉換與提示：

- 孵化後的角色可以升級：同一條養分線持續深化，例如書記型角色逐步成長為森林書記、年輪守望者。
- 孵化後也可以轉換或長出特徵：若後續其他養分變強，角色可轉職或出現混合特性。
- 未孵化與未進化階段不直接列完整條件，可用剪影、傳聞、聲音、氣味或短提示維持期待感。
- 範例提示：
  - 「蛋殼裡傳來紙頁翻動的聲音。」
  - 「蛋殼邊緣有微微草葉香。」
  - 「牠似乎在等待下一次月度整理。」
  - 「港口的風還不夠穩。」
  - 「果園還缺第一場收成。」

孵化生命、守護獸與怪獸的分工：

- 孵化生命：從島嶼蛋誕生的小型生命型態，代表日常習慣、區域狀態、重要事件。名稱、物種、美術都先保留彈性，不預設成人形社群角色。
- 守護獸：大型區域里程碑，代表某條軸線進入成熟階段。
- 怪獸：保留給未來更大型、較有戲劇性的財務目標或長期挑戰，不急著放進第一版。
- 島史：時間線與回憶錄，負責保存「這座島如何走到現在」。
- 圖鑑不是第一個入口。第一版應先呈現島嶼蛋與養分狀態，生命型態 / 守護獸圖鑑在孵化後自然展開。

每個區域先設 3 級進度，不一次塞太多名稱：

| 區域 | Lv1 | Lv2 | Lv3 |
|---|---|---|---|
| 起始海灘 | 完成初始建檔 | 查看並收進第一段島史片段 | 島史橫跨 2 個以上年份 |
| 記帳森林 | 完成建檔後第一個新月份有整理 | 連續或累積 3 個月份有整理 | 一年內 4 個季度都有整理 |
| 成長山丘 | 淨值相對 baseline 小幅成長 | 淨值相對 baseline 穩定成長 | 解鎖成長山丘守護獸 |
| 配置港口 | 有清楚分類的資產 | 有 2 種以上資產大類或使用再平衡 | 配置狀態穩定並解鎖港口守護獸 |
| 收成果園 | 第一筆配息 / 利息 / 租金 | 累積多筆現金流事件 | 累積現金流達階段門檻 |
| 減壓溫泉 | 有負債資料且被納入管理 | 任一筆負債下降到某階段 | 單筆清償或家庭總負債歸零 |

防偏重與防濫用規則：

- 同一天多筆事件可以進島史，但紀錄軸最多算一次整理。
- 同一月份多次新增，不應讓月度 / 季度習慣線暴衝。
- 交易次數不直接給主要進度；完整記錄成本、賣出結果、配置狀態才有意義。
- 配息 / 利息是收成果園路線，不是整個島嶼主線必修。
- 資產多元化只看結構逐漸清楚，不鼓勵為了解鎖而買不需要的資產。
- 淨值成長用相對 baseline，不用絕對資產規模比較使用者。
- 更正舊資料不應刪除既有孵化結果；未來若做重算，也要先設計溫和的回溯規則。

第一階段實作策略：

1. 先完成底層資料模板：`ISLAND_ZONES`、`EGG_STATE`、`NUTRIENT_TYPES`、孵化方向、區域等級、組合事件。
2. 第一版入口先做島嶼蛋與養分狀態，不先做完整圖鑑。
3. 先挑 2 到 3 條最穩的軸線接入養分：起始海灘、記帳森林、成長山丘。
4. 配置港口、收成果園、減壓溫泉先保留資料模板，等前面流程順了再接。
5. 孵化後再展開生命型態 / 守護獸圖鑑，且名字、美術、物種都可後改；底層穩定 ID 與養分規則先穩。
6. 美術、角色圖、動畫、島嶼場景晚一點再做，避免規則還沒穩就重畫。
